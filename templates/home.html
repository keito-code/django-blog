主要なAPIエンドポイント
エンドポイント 説明
/api/v1/blog/posts/ ブログ記事のCRUD操作。
（例: ?search=キーワード で検索も可能です）
/api/v1/blog/token/ JWT認証トークンの取得
/api/v1/blog/token/refresh/ JWTトークンのリフレッシュ
API クイックスタート
JWT認証の使い方
  1. トークンの取得 
    POST /api/v1/blog/token/
Content-Type: application/json

{
    "username": "your_username",
    "password": "your_password"
}
  2. APIリクエストでトークンを使用 
    GET /api/v1/blog/posts/
Authorization: Bearer <your_access_token>
  3. トークンのリフレッシュ 
    POST /api/v1/blog/token/refresh/
Content-Type: application/json

{
    "refresh": "<your_refresh_token>"
}
開発者向け情報
  • このAPIはDjango REST Frameworkで構築されています
  • 認証にはJWT (JSON Web Token) を使用しています
  • 詳細なAPI仕様はSwagger UIまたはReDocでご確認ください
  • フロントエンド開発者の方は、上記のAPIエンドポイントを使用してアプリケーションを構築してください
